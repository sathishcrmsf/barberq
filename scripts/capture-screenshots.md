# Screenshot Capture Guide

This document outlines the screenshots needed for the landing page and marketing materials.

## Required Screenshots

### 1. Dashboard Overview
- **Path**: `/dashboard`
- **Description**: Main dashboard showing KPIs, revenue, queue status
- **Key Elements**: Revenue card, queue count, main actions
- **Size**: 1200x800px (desktop), 375x667px (mobile)

### 2. Queue Management Screen
- **Path**: `/queue`
- **Description**: Active queue with customers in different statuses
- **Key Elements**: Waiting customers, in-progress, completed
- **Size**: 1200x800px (desktop), 375x667px (mobile)

### 3. Add Customer Form
- **Path**: `/add`
- **Description**: Form for adding new walk-in customers
- **Key Elements**: Name field, service dropdown, notes
- **Size**: 1200x800px (desktop), 375x667px (mobile)

### 4. Analytics View
- **Path**: `/analytics`
- **Description**: Revenue charts and performance metrics
- **Key Elements**: Charts, revenue trends, service breakdown
- **Size**: 1200x800px (desktop), 375x667px (mobile)

### 5. Mobile View (Queue)
- **Path**: `/queue` (mobile viewport)
- **Description**: Mobile-optimized queue view
- **Key Elements**: Touch-friendly buttons, mobile layout
- **Size**: 375x667px (iPhone standard)

## Capture Instructions

### Using Browser DevTools
1. Open the app in Chrome/Firefox
2. Open DevTools (F12)
3. Toggle device toolbar (Ctrl+Shift+M / Cmd+Shift+M)
4. Set viewport to desired size
5. Navigate to the page
6. Use browser screenshot or extension

### Using Browser Extensions
- **Chrome**: "Full Page Screen Capture" or "Awesome Screenshot"
- **Firefox**: "FireShot" or built-in screenshot tool

### Manual Steps
1. Navigate to each page listed above
2. Ensure page is fully loaded
3. Populate with sample data if needed
4. Capture screenshot
5. Save to `public/screenshots/` with descriptive name:
   - `dashboard-overview.png`
   - `queue-management.png`
   - `add-customer-form.png`
   - `analytics-view.png`
   - `mobile-queue.png`

## Image Optimization

After capturing:
1. Optimize images using tools like:
   - TinyPNG (https://tinypng.com)
   - ImageOptim (Mac)
   - Squoosh (https://squoosh.app)
2. Convert to WebP format for better performance
3. Ensure file sizes are < 200KB each
4. Maintain aspect ratios
